Continuous Integration/Continuous Deployment (CI/CD) Pipeline

Project Description

This project implements a Continuous Integration/Continuous Deployment (CI/CD) pipeline for a Node.js application built with Express.js. The application was generated using the Express generator, providing a robust foundation for web development. The pipeline automates the build, test, and deployment processes using Jenkins, Docker, and GitHub, ensuring efficient and reliable software delivery.

Main Objective

The objective of this project is to create a fully automated CI/CD pipeline that:

  • Builds the Express.js app whenever new code is pushed to GitHub.
  • Tests the app to ensure quality and stability.
  • Automatically deploys the app using Docker to a test or production environment.

How the Tech Stack Works Together

  1. Code Development(GitHub)
    • Developers push changes (e.g, new features, bug fixes) to the Github repository.
  2. CI Process(Jenkins):
    • Jennkins triggers the pipeline upon receiving a Github webhook
    • Build: Jenkins clones the repository and builds the app according to the Dockerfile, creating a Docker image.
    • Test: Jenkins runs automated tests (e.g, npm test). If tests fail, the pipeline stops and notifications are sent.
    • Deploy: If tests pass, Jenkins pushes the Docker image to a container registry and deploys the container to a target environment.

Monitoring and Notifications

  • Jenkins will be configured to send notifications regarding build success or failure.
  • Logs from Jenkins and Docker containers will provide insight into the build, test, and deployment processes.
